# Git趣文章紀錄規則

所有文章一律放在 `content/projects/*.md`。檔名使用小寫英文、數字與連字號，例如 `pyjokes.md`、`local-ai-runner.md`。

## 網站名稱

網站名稱固定為「Git趣」。前端不顯示紀錄規則與 Markdown 模板，這些檔案只給維護者與 agent 使用。

## 語言規則

- 全文必須使用繁體中文，專案名稱、程式語言、品牌名、網址、指令與必要技術名詞可以保留原文。
- 不使用簡體字。常見禁止字包含：项目、开发、测试、云、数据、应用、用户、环境、支持、实现、体验、优化。
- 內容要像心得文章，不要只填資料。每個判斷都要附上原因或實測結果。
- 第一個段落「這是什麼」會被網站拿來產生摘要，請控制在 2 到 4 段，句子要好讀。

## 必填開頭

每篇文章開頭必須符合以下格式：

```markdown
# 📦 GitHub 專案心得：專案名稱
### 用一句短標語說明這個專案的有趣之處

**日期：** 2026-05-14
**Repo：** [owner/repo](https://github.com/owner/repo)
**⭐ Stars：** ~1k
**語言：** TypeScript
**安裝：** `npm install package-name`
```

## 必填段落

二級標題必須包含以下段落：

```markdown
## 📖 這是什麼
## ⚙️ 原理
## 🧪 測試結果
## 💡 想法與心得
## 📊 總評
```

可以在每個二級標題底下使用三級標題、表格、程式碼區塊與清單，但不要新增其他二級標題。

## 網站資料抽取規則

`scripts/build-data.mjs` 會自動從 Markdown 抽出：

| 網站欄位 | 來源 |
| --- | --- |
| 文章標題 | `# 📦 GitHub 專案心得：...` |
| 副標 | `### ...` |
| 更新日期 | `**日期：**` |
| GitHub 連結 | `**Repo：**` |
| Stars | `**⭐ Stars：**` |
| 語言 | `**語言：**` |
| 安裝 | `**安裝：**` |
| 摘要 | `## 📖 這是什麼` 第一段內容 |
| Tags | 腳本依語言與內文關鍵字推定 |
| 推薦文章 | 心得文格式預設進入推薦候選 |

## 品質檢查

提交前必須執行：

```bash
npm run validate
npm run build:data
```

`data/projects.json` 是網站讀取資料，不要手動編輯。
