## 📖 這是什麼

pyPipes 是一個**終端機水管跑馬燈**，在終端畫面中不斷生成由 Unicode 框線字元組成的「水管」圖案，模擬 Windows 95 時代經典的「水管」螢幕保護程式。

它沒有任何參數可以設定，就是一行命令跑到底，按下 Enter 鍵才會停止。說穿了就是一個 `while True` 迴圈加上 `random.choice()` 配合 Unicode 框線字元。

---

## ⚙️ 原理

```
上游（up）    = "╚╩╠╬╣║╝"
下游（down）  = "╔╦╠╬╣║╗"
左側（left）  = "╩╦═╬╣╗╝"
右側（right） = "╚╔╩╦╠═╬"
        ↓
generate_row(length)
  → 每一個字元根據「上方是否需要連接」與「左方是否需要連接」
    來決定可用的水管字元集合
  → 隨機挑選一個
        ↓
   輸出完整一行水管字元
```

**核心邏輯：** 每個水管字元是否「開放」某個方向，決定了下一個字元必須是什麼形狀，才能保持水管連貫。

**停止機制：** `select.select()` 監聽 stdin，若讀到 Enter 就 `break`。

### 6. 顏色支援

**原始版本沒有任何顏色支援。** 所有輸出都是終端預設字體顏色（通常為白色或綠色）。

如果要彩色版本，需要自己用 ANSI escape codes 包裝，例如：

```python
# 示範彩色水管
COLOR = '\033[92m'  # 綠色
RESET = '\033[0m'
print(COLOR + new_row + RESET)
```

## 💡 想法與心得

### 這個專案教會我的事

**1. 最小可行產品的經典範例（又一次）**

和 pyjokes 一樣，pyPipes 也是那種「一行命令就能跑」的最小工具。沒有參數、沒有配置、沒有依賴，就是 `python pyPipes.py`。這種設計哲學在 CLI 工具中非常常見。

**2. Unicode 框線字元的藝術**

這支程式用了 8 個上游、8 個下游、8 個左側、8 個右側共 32 個 Unicode 框線字元，來實現水管的「視覺連貫」。這個技巧可以用在其他需要方塊字元的場景。

**3. 與經典 Windows 水管螢幕保護程式的差距**

pyPipes 只在同一列生成水管，沒有「水流方向」的動畫效果。經典的 Windows 水管螢幕保護程式水管會「彎曲」，而且有動畫效果。pyPipes 更像是「靜態水管一字排開」而不是真正的水管。

**4. Python 2 語法相容性問題**

原始程式碼使用 `print new_row`（Python 2 語法）和 `unicode()` 函式，這在 Python 3 中完全無法執行。需要手動修改才能在現代環境中運行。

---

### 實用場景

| 場景 | 怎麼用 |
|------|--------|
| 終端機開機畫面 | 缺點：這程式會 block，需要背景執行 |
| 螢幕保護程式替代 | 適合 Linux 無頭（headless）伺服器 |
| 螢幕錄製素材 | 慢速模式 + 固定視窗大小，錄製效果很不錯 |
| 教學範例 | 展示 Unicode 字元、字串處理、連貫性邏輯 |

---

### 已知問題

1. **`old_row` 索引錯誤：** 當 `generate_row()` 的長度大於 60 時，`old_row[pos]` 可能超出範圍導致 `IndexError`。這是原始程式碼的 bug，長度越大越容易觸發。

2. **Python 3 不相容：** `print` 語法與 `unicode()` 函式都已經在 Python 3 中移除，需要修改原始碼。

3. **沒有顏色支援：** 無法自訂顏色或使用 ANSI 色彩。

4. **沒有長度/延遲參數：** 必須修改程式碼才能調整水管長度或刷新速度。

## 🔧 修正後的 Python 3 版本

原始 `pyPipes.py` 有 Python 2 語法問題，以下是修正摘要：

1. `u"..."` → `"..."`（Python 3 預設即為 Unicode）
2. `print new_row` → `print(new_row)`
3. `unicode()` → `str()`
4. `old_row[pos]` 加入長度檢查，防止 `IndexError`

修正後可在 Python 3.12 正常執行。
